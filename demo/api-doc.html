JavaScript API

Beyond the declarative attributes, the <wayfinder-map> element exposes an imperative API on its DOM instance. From host JavaScript you can trigger a route, focus a shop, switch floors, drive the camera, read the catalog, and listen for events — everything the built-in controls do, under your own UI.

Open the demo →

When to use this pattern

Drive the map from JavaScript when you want your own chrome instead of (or alongside) the built-in controls — an external search box that calls focusLocation(), a “Get directions” button that calls navigateTo(), a deep-link handler that switches floors and zooms on load, or analytics that react to location-tap events. The element still renders the map; you orchestrate it.

Step-by-step setup

1. Import and grab the element

Same import boilerplate as every demo: an import map for qrcode-generator plus the module import that registers the custom element. 

<wayfinder-map data-url="../datas/SGC_v001.json" default-floor="L3" level-selector></wayfinder-map>

<script type="module">
  import "../dist/wayfinder-map.esm.js";
  const map = document.querySelector("wayfinder-map");
</script>

2. Wait for ready before calling methods

init() runs automatically on connect when data-url is present, but it is asynchronous (it fetches the bundle). Calling a method before the engine is ready throws "wayfinder-map: Not initialized". Wait for the ready event — and guard against it having already fired. 

function start() {
  // safe to call the API now
  console.log(map.floors, map.currentFloor);
}
if (map.isInitialized) start();
else map.addEventListener("ready", start, { once: true });

3. Resolve IDs from getLocations()

Location IDs are namespaced strings like "shop:10" — not raw numbers. (The focus-shop-id attribute takes a number and prefixes it for you; the JS API does not.) Always source IDs from getLocations() and pass them straight through. Only placed locations are routable, so filter on displayNodes. 

const routable = map.getLocations()
  .filter((loc) => loc.displayNodes?.length > 0 && loc.levelCodes?.length > 0);
// each item: { id: "shop:10", label: "Starbucks", levelCodes: ["L3"], ... }

Seed data note: the bundled SGC_v001.json places only 6 locations — five on L3 (shop:1, shop:4, shop:7, shop:10, shop:11) and one on L2 (shop:14). Routing succeeds between placed locations; other catalog entries have no placement and will fail to route or focus.

4. Trigger a route — branch on result.success

navigateTo() never throws for an un-routable pair — it returns a result object. Check success and read code on failure. A route-found / route-error event also fires. 

const result = map.navigateTo({ from: "shop:10", to: "shop:1", mode: "escalator", animate: true });
if (result.success) {
  console.log("route", result.startLocation.title, "→", result.endLocation.title, result.distance);
} else {
  console.warn("no route:", result.code, result.error);
}

// or just focus a single shop, no route:
map.focusLocation("shop:4", { animate: true });

Methods

MethodDescription
init()Boot the engine. Auto-runs on connect when data-url is set; returns a Promise.
navigateTo({ from, to, mode?, animate?, duration?, scale?, connectorConstraint?, stepFree? })Route between two location IDs. Returns a result (see below); dispatches route-found / route-error.
focusLocation(id, { clearRoute?, switchFloor?, animate?, duration?, scale? })Pin & zoom a single location (no route). Returns { success, location, node, floor }; dispatches location-tap.
clearRoute()Return to browse mode, remove route/pin. Dispatches route-cleared.
setFloor(code)Switch floor, holding zoom/pan/rotation. Dispatches floor-changed.
resetView()Reset zoom, pan, and rotation to the initial framing.
resetRotation()Reset rotation to 0° without changing zoom/pan.
centerOn(x, y, options?)Pan (and optionally animate/zoom) to a world coordinate.
zoom(factor)Relative zoom about the centre — 1.2 in, 0.8 out.
getLocations()The searchable catalog — array of location objects (id, label, levelCodes, displayNodes, …).
getLocation(id)Look up one location by its namespaced ID, or undefined.
getViewState()Current camera as { scale, panX, panY, rotation }.

navigateTo() returns { success, startLocation, endLocation, distance, segments, transitions, startAnchor, endAnchor } on success, or { success: false, code, error } on failure.

Properties

PropertyAccessDescription
currentFloorget / setActive floor code; setting it switches floor (same as setFloor).
routeModeget / set'escalator' or 'lift' — the vertical-connector preference.
floorsgetArray of floor codes, e.g. ["B2","B1","L1","L2","L3"].
levelsgetArray of level objects with metadata (code, name, position).
hasRoutegetWhether a route is currently displayed.
isInitializedgetWhether init() has completed — gate API calls on this.
enginegetThe internal MapEngine — an advanced escape hatch; prefer the methods above.

Events

All events bubble and cross the shadow boundary (composed: true). Listen with addEventListener; read the payload from event.detail. 

map.addEventListener("route-found", (e) => console.log("route", e.detail));
map.addEventListener("location-tap", (e) => console.log("tapped", e.detail.location));
map.addEventListener("floor-changed", (e) => console.log("floor", e.detail));
Eventdetail
ready— engine initialized, API safe to call.
error{ error } — init or operation failure.
data-loaded— the bundle finished loading.
floor-changed{ floor } — new floor code.
view-changed{ scale, panX, panY, rotation }.
route-found{ startLocation, endLocation, … }.
route-error{ error, from, to }.
route-cleared— returned to browse mode.
route-mode-changed{ mode } — 'escalator' / 'lift'.
location-tap{ location, node, floor } — a unit was tapped/focused.
floor-transition-tap{ transitionId, fromFloor, toFloor } — a connector bubble was tapped.
floor-tap{ floor, x, y } — empty floor area.
location-disambiguate{ locations, x, y } — a multi-tenant unit needs a choice.
empty-tap{ x, y } — empty map area.

Minimal runnable example

The essential wiring, without the demo’s control-panel chrome. See api.html for the full styled page. 

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1,
          maximum-scale=1, user-scalable=no" />
    <title>Saigon Centre Map</title>
    <style>
      html, body { margin: 0; height: 100%; overflow: hidden; }
      wayfinder-map { display: block; width: 100vw; height: 100vh; }
    </style>
    <script type="importmap">
      { "imports": { "qrcode-generator": "../qa-shims/qrcode-generator.js" } }
    </script>
  </head>
  <body>
    <wayfinder-map
      data-url="../datas/SGC_v001.json"
      default-floor="L3"
      level-selector
    ></wayfinder-map>

    <script type="module">
      import "../dist/wayfinder-map.esm.js";
      const map = document.querySelector("wayfinder-map");
      map.addEventListener("error", (e) => console.error("Map error:", e.detail?.error || e));

      function start() {
        // pick two placed shops from the catalog
        const routable = map.getLocations()
          .filter((loc) => loc.displayNodes?.length > 0);
        if (routable.length >= 2) {
          const result = map.navigateTo({
            from: routable[0].id,   // e.g. "shop:10"
            to: routable[1].id,     // e.g. "shop:1"
            animate: true,
          });
          if (!result.success) console.warn("no route:", result.code);
        }
      }
      if (map.isInitialized) start();
      else map.addEventListener("ready", start, { once: true });
    </script>
  </body>
</html>
